Problem & Objectives

What is 3DAgent?

An embodied, conversational AI agent built on Matterport digital twins. It fuses large language models, multimodal computer vision, and the Matterport SDK to turn passive 360° tours into spaces you can talk to, scan, navigate, and maintain — automating asset cataloguing and facility audits for non-technical users.

Problem & Objectives

Traditional 3D spaces are passive and disconnected; facilities teams spend hours cataloguing assets by hand from floor plans. 3DAgent maps natural-language intent to action, auto-catalogues assets from a screenshot, and closes the loop: detect → tag → query → report → triage → resolve.

Key Features
🗣️

NL & Voice Navigation

"I want to cook" or "take me to the nearest fire extinguisher" — Web Speech API enables hands-free control.

📷

AI Vision Asset Detection

Scout + YOLOv8-seg + Grounding DINO detect, count, classify, outline, and auto-tag assets from a viewport.

ReAct Agentic Verification

Novelty & Inventiveness
  1. Digital Twin × LLM fusion — a 70B LLM drives a real Matterport camera, so it teleports on reasoning, not menu clicks; one chat box becomes a 14-class semantic router.
  2. Hybrid open-vocabulary vision — Llama 4 Scout names any item (open vocab), then YOLOv8-seg + Grounding DINO ground each name with boxes and edge-fitting polygon outlines — more reliable than asking an LLM for coordinates.
  3. ReAct agentic verification — sense → think → act → verify: the agent visits each candidate room and confirms its current state with live vision before answering.
  4. Voice-filed, location-pinned maintenance — "report chair #1 has a broken leg" extracts asset + severity and pins the issue to an exact sweep for later fly-to repair.
  5. Resilient by design — cloud Groq inference for speed, with a local BLIP VQA + YOLO path that keeps detection working with no cloud key.
Usefulness & Application

🎯 Who & what it replaces

Built for facilities & operations managers, factory/warehouse supervisors, lab & hospital asset coordinators, and real-estate / retail teams. Replaces clipboard audits, static PDF floor plans, spreadsheet registers, and disconnected maintenance tickets — hours of cataloguing collapse into a few screenshots.

🖥️ Hardware Specification

  • Capture: Matterport Pro2 / Pro3 (or compatible 360°/LiDAR camera) to build the digital twin.
  • Server / dev: i7 / Ryzen 7 · 16 GB RAM · NVIDIA CUDA GPU ≥ 6 GB VRAM for fast local CV (CPU-only also works) · ~5 GB for model weights · Python 3.10+.
  • Cloud: Groq API serves LLaMA 3.3 70B + Scout 17B — no local GPU needed for the LLM/VLM path.
  • Client: any modern browser (Chrome/Edge) with WebGL + microphone.
